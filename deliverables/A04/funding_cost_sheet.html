Scaling approach

We set up one VM with Rhino + our custom MCP server, capture a golden image, then clone it to spin up parallel instances on demand. Each clone connects back to Claude via SSH tunnel and our routing server (already built and tested locally). VMs are deallocated when idle — compute costs stop immediately, only storage continues.

Risk mitigation

  • Phased cost gates — each phase requires the previous to succeed. No bulk commitment.
  • Dominant cost is Claude Max (200 CHF/month) — already running, already proven. New infrastructure adds only 100–160 CHF/month at full capacity.
  • Free tiers cover secondary tools — Gemini credits, Blender, GitHub Pages all at zero cost.
  • Golden image approach — set up one VM, clone as needed, deallocate when idle. No wasted compute.
  • EPFL credits — 100 CHF Azure offset, no credit card required.
  • Everything built by the team — multi-instance MCP routing, SSH tunneling, agent coordination, knowledge base — all developed and tested in-house.
  • Rhino license retains value — if cloud approach fails, the license works locally on any Windows machine.
