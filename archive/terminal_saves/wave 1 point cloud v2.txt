  The key question: free pipeline vs paid alternatives

  The research was done assuming CHF 0. Now that Huang offered tool support,
  let's reconsider.

  What paid options actually exist?

  Grasshopper plugins that could help:
  - Elk / Heron — these import GIS data (shapefiles, OSM, rasters) into
  Grasshopper. Heron is free. Elk is free. They'd help with the swissALTI3D →
  terrain mesh path but don't handle point clouds.
  - Lands Design (~CHF 500/yr) — landscape/terrain tools, but overkill here.
  - Urbano — urban analysis, not really point cloud processing.
  - RhinoTerrain (~€800) — dedicated terrain from point clouds/contours. This is
   the one paid tool that could genuinely compress the pipeline — it does point
  cloud → terrain mesh directly inside Rhino.

  Cloud processing:
  - Services like Pointly, PointCab, or Potree are more for visualization than
  producing Rhino-ready meshes.

  My honest assessment

  The free pipeline (PDAL → CloudCompare → Rhino) is actually the right call
  even with budget. Here's why:

  1. The data is already free and pre-classified. The biggest cost saving is
  already baked in — swisstopo OGD eliminates the survey step entirely.
  2. The bottleneck isn't the tools, it's the meshing step. And CloudCompare's
  Poisson reconstruction is as good as any paid alternative for LOD 100-200.
  3. PDAL is industry standard. Learning it is more valuable than learning a
  paid plugin that wraps the same operations.
  4. The "quick path" fallback is strong. swissALTI3D heightfield + swissTLM3D
  extruded footprints can be done in an afternoon per site. For a BA6 studio at
  LOD 100-200, this might actually be sufficient.

  Where I'd consider spending money

  The only place budget might help:
  - RhinoTerrain if the CloudCompare → Rhino handoff proves painful for 7 sites
  - A better machine / more RAM if point cloud processing bogs down (but this is
   probably fine for 1km² tiles)

  What I'd actually recommend

  Start with the quick path (Section 10) for all 7 sites first — swissALTI3D
  terrain + swissTLM3D building massing. Get working site models in Rhino within
   1-2 days. Then selectively upgrade to the full point cloud pipeline for the
  2-3 sites where you need vegetation canopy or precise roof geometry for the
  intervention design.

  This avoids the ~15-20 hour pipeline build before you have any site model in
  Rhino. You get something usable fast, then invest in quality where it matters.

  One thing missing from the research

  swissBUILDINGS3D 2.0 — mentioned briefly but not explored. It's free (OGD),
  gives LOD2 buildings in CityGML/DXF, and might be a better building source
  than either point cloud extrusion or swissTLM3D footprints. Worth a quick
  check on whether it covers your 7 sites before committing to the point cloud
  path for buildings.

  ---
  What's your read? Do you want to go quick-path-first, or does the full
  pipeline feel worth investing in upfront?
