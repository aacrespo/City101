│ PART 2: REVIEW PANEL PROTOCOL                                               │
│                                                                             │
│ Format                                                                      │
│                                                                             │
│ After each LOG stage (300, 350, 400), the modeler pauses and the review     │
│ panel runs. The panel is 5 separate agents spawned as a team — they can     │
│ interact, challenge each other, and surface tensions between their domains  │
│ (e.g., structural says "add column" but accessibility says "that blocks the │
│  turning radius"). This inter-agent dialogue is where the real value        │
│ emerges.                                                                    │
│                                                                             │
│ All 5 reviewers are permanent agents in .claude/agents/ — reusable for any  │
│ future lock, building, or architectural review.                             │
│                                                                             │
│ Input to Review Panel                                                       │
│                                                                             │
│ The modeler provides:                                                       │
│ 1. Viewport screenshots (4 views: perspective, plan, section, detail)       │
│ 2. Layer tree listing                                                       │
│ 3. Geometry summary (object count per layer, bounding box, key dimensions)  │
│ 4. Current LOG target and what was just added                               │
│ 5. The lock's concept description (from prototypology_content.json)         │
│                                                                             │
│ The 5 Lenses                                                                │
│                                                                             │
│ Lens 1: Concept Critic                                                      │
│ - Question: "Does this still read as a lock/threshold?"                     │
│ - Checks: Two states visually distinct? Gate identifiable from any angle?   │
│ Chamber program legible?                                                    │
│ - PASS if: A stranger could identify the two states and the transition      │
│ mechanism                                                                   │
│ - FAIL example: "The gate merges visually with the night chamber — no       │
│ threshold legible"                                                          │
│                                                                             │
│ Lens 2: Structural Reviewer                                                 │
│ - Question: "Could this stand up?"                                          │
│ - Checks: Continuous load paths? Column-beam connections logical? Spans     │
│ appropriate for material? Slab depths reasonable?                           │
│ - Reference: Material-span rules (timber ≤8m, steel ≤15m, concrete ≤12m for │
│  simple spans)                                                              │
│ - PASS if: Load can be traced from roof to ground without interruption      │
│ - FAIL example: "Cantilevered slab at 6m with no visible support — add      │
│ column or reduce span"                                                      │
│                                                                             │
│ Lens 3: Envelope Reviewer                                                   │
│ - Question: "Does the skin make sense?"                                     │
│ - Checks: Mullion grid regular? Material transitions at logical breaks?     │
│ Weathering details (drip edges, flashings at LOG 400)? Glass-to-opaque      │
│ ratios buildable?                                                           │
│ - PASS if: A façade consultant wouldn't flag major issues                   │
│ - FAIL example: "Glass wraps around corner without a mullion — add corner   │
│ post or detail the connection"                                              │
│                                                                             │
│ Lens 4: Accessibility Reviewer                                              │
│ - Question: "Can everyone use this?"                                        │
│ - Checks: Ramp grades ≤6% (SIA 500)? Doors ≥900mm clear? Level thresholds   │
│ at entries? Turning radii ≥1500mm? Tactile guidance paths? Elevator present │
│  if multi-story?                                                            │
│ - Reference: Swiss SIA 500 norms                                            │
│ - PASS if: A wheelchair user could navigate the entire building             │
│ independently                                                               │
│ - FAIL example: "Ramp between chambers is 8% grade — reduce to 6% max or    │
│ add intermediate landing"                                                   │
│                                                                             │
│ Lens 5: LOG Compliance                                                      │
│ - Question: "Is this the right level of detail?"                            │
│ - Reference: 00_Workflow_v04.md Section 3.2 LOG table                       │
│ - At LOG 300: façade divisions, structural elements, all openings present?  │
│ - At LOG 350: mullions, railings, stairs with runs/landings, key details?   │
│ - At LOG 400: material breaks, connections, hardware, panel joints?         │
│ - PASS if: All elements listed for the target LOG level are present         │
│ - FAIL example: "LOG 400 target but no panel joints on curtain wall — add   │
│ joint lines at 1200mm module"                                               │
│                                                                             │
│ Output Format                                                               │
│                                                                             │
│ ## Review: Lock XX — LOG YYY                                                │
│                                                                             │
│ ### Concept Critic: [PASS/FAIL]                                             │
│ [1-2 sentence assessment]                                                   │
│ [If FAIL: specific fix instruction]                                         │
│                                                                             │
│ ### Structural: [PASS/FAIL]                                                 │
│ [1-2 sentence assessment]                                                   │
│ [If FAIL: specific fix instruction with dimensions]                         │
│                                                                             │
│ ### Envelope: [PASS/FAIL]                                                   │
│ [1-2 sentence assessment]                                                   │
│ [If FAIL: specific fix instruction]                                         │
│                                                                             │
│ ### Accessibility: [PASS/FAIL]                                              │
│ [1-2 sentence assessment with measurements]                                 │
│ [If FAIL: specific fix instruction with SIA 500 reference]                  │
│                                                                             │

│ Iteration Rules                                                             │
│                                                                             │
│ - All 5 PASS → advance to next LOG stage                                    │

│ - Critical vs suggested: only critical fixes block advancement              │
