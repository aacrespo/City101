⏺ Blender MCP Recommendation

  Winner: https://github.com/ahujasid/blender-mcp — 18K stars, actively
  maintained, install via uvx blender-mcp.

  Why it wins for you: built-in asset pipeline. Poly Haven (textures, HDRIs,
  furniture), Sketchfab (search + download 3D models), and AI-generated assets
  (Hyper3D). For furnishing interiors and populating scenes, this is the only
  one that doesn't require you to source everything manually. Plus
  execute_blender_code lets Claude write any bpy Python for animation, camera
  paths, rendering — so the 22 named tools are the floor, not the ceiling.

  Rhino geometry comes in via OBJ/FBX export → bpy import. Same pattern as your
  Rhino MCP — named tools for common ops, code execution for everything else.

  ---
  Now your follow-up questions:

  Other MCPs — yes, that's covered in Prompt 2 (Part 5: Tool & Workflow
  Scaling). The vision roadmap session will research the full MCP landscape.

  Blender vs Rhino for architecture drawings — stick with Rhino for design +
  drawings. Rhino is purpose-built for precision: exact dimensions, construction
   layers, 2D drafting (Make2D), clean sections. Blender is a mesh modeler —
  great for visuals, terrible for measured drawings. The right split is:
  - Rhino: design, construction detail, plans/sections/elevations, precise
  geometry
  - Blender: rendering, animation, texturing, interior experience, presentation
  visuals

  Trying to do construction drawings in Blender would be fighting the tool. The
  transfer step (Rhino → OBJ → Blender) is worth it.

  Blender animations on interactive websites — yes and no. You have options:

  1. Pre-rendered video — render in Blender, embed as <video> on any website.
  Simplest.
  2. glTF/GLB export — export from Blender to glTF format, then load in three.js
   on a webpage. This gives you interactive 3D (orbit, zoom, walk) in the
  browser. This is what most architecture firms do for web presentations.
  3. Blender → USD → web viewer — newer pipeline, same idea.
  4. Sketchfab embed — upload your Blender model to Sketchfab, get an embed
  iframe. Zero code.

  So it's not just animation — you can have fully interactive 3D on your
  scrollytelling site. Three.js + glTF is the standard pipeline for that, and
  you already have a site infrastructure (visualizations/site/).

  Three answers:

  1. Pixel Agent Dashboard — Tracking Teammates (not just subagents)

  Yes, it's possible. The pixel-agents codebase already tracks sessions via
  JSONL file watching. The key difference:

  - Subagents write to the same JSONL file as the parent (they're in-process).
  Easy to detect — the code already does it.
  - Teammates (Agent Teams) run as separate sessions with their own JSONL files
  in ~/.claude/teams/{team-name}/. They're independent processes.

  So you'd need the file watcher to also watch ~/.claude/teams/ for new session
  files appearing. Each teammate gets its own character on the dashboard. The
  data contract is the same (JSONL with tool_use, tool_result events) — it's
  just multiple files instead of one.

  The coordination messages between teammates (when one sends a message to
  another) would show up as speech bubbles between the pixel characters. That's
  the cool part — you'd literally see the organigram in action.

  Short answer: Yes, ~2-3 hours of extra watcher code on top of the existing
  subagent tracking.

  ---
  2. Blender MCP APIs — What They Are & What They Cost

  From the research earlier, the blender-mcp server has integrations with four
  external services:

  Service: Poly Haven
  What It Does: Free library of textures, HDRIs, and 3D models. CC0 licensed.
    Thousands of materials (concrete, wood, brick, metal — perfect for
    architecture).
  Cost: Free. No API key needed.
  Need It?: YES — this is your main texture/material source. Concrete walls,
  wood
     floors, brick, metal — all here for free.
  ────────────────────────────────────────
  Service: Sketchfab
  What It Does: Search and download 3D models (furniture, people, vegetation,
    vehicles). Huge library.
  Cost: Free tier with API key (limited downloads). Paid plans exist but free is

    enough. API key from sketchfab.com/settings/password.
  Need It?: YES for later — when you need to populate scenes with furniture,
    people, trees. Not needed for the blob animations tomorrow.
  ────────────────────────────────────────
  Service: Hyper3D Rodin
  What It Does: AI-generated 3D models from text or images. "A wooden chair" →
  3D
    mesh.
  Cost: Free tier available (limited). Paid beyond that.
  Need It?: Maybe — interesting for generating quick architectural props from
    description. But quality is hit-or-miss for architecture. Try later.
  ────────────────────────────────────────
  Service: Hunyuan3D (Tencent)
  What It Does: Same idea — AI text/image to 3D model. Tencent's version.
  Cost: Free tier available.
  Need It?: Skip for now — similar to Hyper3D, experimental quality.

  For tomorrow's blob animations: You need none of these APIs. The house set and
   blobs are all procedural geometry via execute_blender_code. No textures
