What Swiss academic culture expects in a funding request:
Proportionality. Swiss institutions are careful with money but not stingy when the case is clear. What they hate is vagueness. "I need more API credits" is bad. "I need approximately 150 CHF/month for Gemini embedding API and Claude inference to run the multi-agent system I've built, which produces norm-compliant Rhino models for the studio project" is good. At EPFL, even small amounts need a clear line item justification. Frame it as a tool cost, like buying a license — because that's what it is.
Equity across the cohort. The assistant will immediately think: "If I approve this for Andrea, do I need to approve it for everyone?" Your answer should preempt this. Your system is shared infrastructure — Henna already uses it, the knowledge base is generalizable, the repo is on GitHub. This isn't a personal luxury; it's a tool the studio could adopt. If anything, funding your work benefits multiple students.
Pedagogical alignment. The assistant needs to believe this is making you a better architect, not just a better programmer. Connect every technical piece back to architectural knowledge. The knowledge base isn't about embeddings — it's about ensuring construction details meet Swiss norms. The agent team isn't about multi-agent orchestration — it's about scaling the review process that architects do manually. Always translate the technical into the architectural.
Demonstrable output. Swiss academic culture is pragmatic. They want to see what you've made, not hear about what you plan to make. You have 399 modeled objects, 17 exercises, a working knowledge base, and a research paper. Lead with artifacts, not ambition.
