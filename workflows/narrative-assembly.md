# Workflow: Narrative Assembly

## Objective
Compile findings, data, and visualizations into a coherent narrative deliverable.

## When to use
When assembling narrative sections for submission or presentation.

## Required inputs
- Findings from `observations/INDEX.md`
- Relevant datasets from `datasets/`
- Visualizations from `visualizations/`

## Steps
1. **Read `observations/INDEX.md`** — what findings are confirmed, what needs verification?
2. **Identify the argument structure** — what's the thesis, what evidence supports it?
3. **Reference datasets by path** — don't copy data into narrative, link to it
4. **Reference visualizations by path** — include figure references
5. **Draft in `deliverables/`** — use descriptive filename (e.g., `A02_NARRATIVE_DRAFT_v2.md`)
6. **Review theoretical anchors** — Comtesse (diversity × accessibility × time), Deleuze (surface), fractal principle
7. **Check with Andrea/Henna** before finalizing — narrative direction is a human decision

## Expected output
- Markdown draft in `deliverables/`
- Clear references to all supporting data and visualizations

## Edge cases
- Findings contradict each other: flag the tension, don't resolve it without human input
- Missing data for a claim: note the gap, suggest what data would fill it
